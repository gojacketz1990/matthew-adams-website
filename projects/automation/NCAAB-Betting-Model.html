Python NCAAB Betting Model

**Category:** Web Automation Framework | **Tools Used:** Selenium WebDriver, Python, GitHub, Pandas, Web Scraping, Data Science | **Duration:** 2 Months

Project Overview

This project involved designing and implementing a robust and scalable test automation framework using Selenium WebDriver with Python. The primary goal was to create a maintainable and reusable suite for an e-commerce platform's critical user journeys, centered around the Page Object Model (POM) design pattern to separate test logic from UI locators.

Process

  1. Define the problem - predict outcomes of NCAA Men's Basketball Games
  2. Identify Data that will be needed
  3. Evaluate the Data
  4. Determine pertinent features
  5. Create a model
  6. Experiment and Test the model
  7. Implement the model
  8. Next Steps

Note: This project was implemented without using Machine Learning and has proven to be successful. The next steps are to implement machine learning algorithms on the data.

Key Challenges

Before implementing the framework, there were challenges to be faced:

  • **Maintainability:** UI changes often broke tests, as locators were hardcoded within test scripts.
  • **Code Duplication:** Common actions and helper methods were replicated across multiple files.
  • **Readability:** Scripts were difficult to read and understand, blending test logic with page interactions.
  • **Inconsistency:** There are many inconsistencies between Kenpom and VegasInsider. Maps of team names had to be created to link some of the smaller schools which were inconsistent.

My Approach & Design

I built the framework from the ground up, with Pytest as the test runner due to its simplicity and powerful features like fixtures. The core of the framework is the Page Object Model, which provides a clean and structured way to manage test code.

Page Object Model Implementation:

Each distinct web page or major component (e.g., login page, checkout page, header) of the application was represented by a separate Python class. This class contained:

  • Web Elements: A list of locators (e.g., `By.ID`, `By.XPATH`) for all the elements on that page.
  • Page Methods: Actions that can be performed on the page (e.g., `login(username, password)`, `click_add_to_cart()`).

This approach ensured that if a UI element's locator changed, only one file (the Page Object) needed to be updated, not every single test script that used it.

Framework Features:

  • **Pytest Fixtures:** Used to manage browser instances and other setup/teardown logic efficiently.
  • **Configuration Management:** A centralized `config.ini` file to handle environment-specific data like URLs, usernames, and passwords.
  • **Test Data Handling:** Utilized CSV files and Python's `unittest.TestCase` to enable data-driven testing for scenarios like form validation and user logins.
  • **Reusable Utilities:** Created a separate `utils` module for common tasks like taking screenshots, handling waits, and logging.

Running the Model

  • **Enhanced Maintainability:** The POM design reduced the effort required for test suite maintenance by approximately 50%.
  • **Improved Code Readability:** Test scripts became concise and easy to understand, reading almost like plain English.
  • **Increased Reliability:** The use of explicit waits and a stable framework structure led to more consistent and reliable test runs.
  • **Reliability:** There are many inconsistencies between Kenpom and VegasInsider. Maps of team names had to be created to link some of the smaller schools which were inconsistent.

The following is an example of a Page Object class for a login page:

Example: Login Page Object (Python)


                  from selenium.webdriver.common.by import By
                  from selenium.webdriver.support.ui import WebDriverWait
                  from selenium.webdriver.support import expected_conditions as EC

                  class LoginPage:
                      def __init__(self, driver):
                          self.driver = driver
                          self.username_input = (By.ID, "user-name")
                          self.password_input = (By.ID, "password")
                          self.login_button = (By.ID, "login-button")

                      def login(self, username, password):
                          WebDriverWait(self.driver, 10).until(
                              EC.visibility_of_element_located(self.username_input)
                          ).send_keys(username)
                          self.driver.find_element(*self.password_input).send_keys(password)
                          self.driver.find_element(*self.login_button).click()
